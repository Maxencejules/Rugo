Full-OS implementation guide Part IV.10 (security): replace the symmetric
keyed-hash package integrity check with a real **public-key** signature.

## Why this is public-key (and the old scheme was not)

The earlier package-integrity check used **HMAC-SHA256**, a *symmetric* keyed
hash: the kernel held the same secret key used to produce the tag, so it could
equally well **forge** a valid tag for arbitrary content. That authenticates
"someone who knows the key", not "the package author".

This uses a genuine **asymmetric** signature. The signer holds a private key
(256 secret preimage pairs) and never shares it; the kernel embeds only the
**public** key (the SHA-256 hashes of those preimages) and a signature. With the
public key alone the kernel can *verify* a signature but cannot *produce* one —
the security property the audit asked for.

## Lamport one-time signature

Parameters: SHA-256 (`n = 32` bytes), a 256-bit message hash.

- **private key** `sk[i][b]` — 256 pairs of 32-byte secret preimages (offline only);
- **public key** `pk[i][b] = SHA256(sk[i][b])` — 256×2×32 = 16384 B (embedded);
- **signature** `sig[i] = sk[i][ bit i of SHA256(msg) ]` — 256×32 = 8192 B;
- **verify**: for each bit `i` of `SHA256(msg)`, check `SHA256(sig[i]) == pk[i][bit]`.

`lamport_verify` always walks all 256 positions and fails on any mismatch. It
reuses the in-kernel SHA-256 (`crate::sha256::sha256`) — no big-integer or
elliptic-curve arithmetic, so the verifier is small and auditable.

The keypair and the one signature over the fixed message `"rugo-pkg-sign-v1"` are
produced deterministically by `tools/lamport_keygen_v1.py` (fixed seed, so the
committed binaries are reproducible). The private key is discarded — only the
public key and the signature are committed/embedded.

## Behaviour

`sys_sigverify(op)` (id 63) exercises the verifier against the embedded key +
signature, returning `1` (accept) / `0` (reject):

- **op 1** — the genuine signature over `"rugo-pkg-sign-v1"` → **1**;
- **op 2** — a tampered message (`"rugo-pkg-sign-vX"`) selects different
  preimages, none of which hash to the public key → **0**;
- **op 3** — a tampered signature (one preimage bit flipped) no longer hashes to
  its public-key entry → **0**.

`sigverify_selftest` runs all three at boot
(`PQSIG: lamport verify ok, forgery rejected`). `pqsigprobe` runs them from ring
3 (`PQSIGAPP: verify ok forge rejected`), so the verifier is exercised on a live
syscall path, not only a boot self-test.

## v1 boundary / carry-forward

- **One-time key.** Lamport keys sign a single message; signing N packages needs
  N keys, a Merkle tree over them (XMSS/SPHINCS+-style), or a few-time scheme.
  The verify primitive and its asymmetry are what is proven here.
- **Not yet wired into the package store.** The 8 KiB signature is large relative
  to the size-tight on-disk package region, so the live `sys_spawn` package load
  still uses its existing integrity check; swapping it for a Merkle-batched
  hash-based signature (to amortise the per-package size) is the follow-up.
- **Message is fixed.** `op` selects fixed embedded vectors rather than copying an
  arbitrary message+signature in from userspace; a general
  `sys_sigverify(msg_ptr, sig_ptr)` is a natural extension once the package store
  carries per-artifact signatures.
